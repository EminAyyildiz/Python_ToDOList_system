# Python_ToDOList_system


# Bu proejdeki temel hedefim hafta içerisinde yapam gereken görevleri bir sisteme eklemek ve o sistemden görebilmek daha sonra da silebilmektir. Tabi ki bu görevi yapabilmek için birçok sistem zaten mevcut. Ancak eğitimimdeki bu uzun serüven boyunca kendi yaptığım sistemi kullanmak daha güzel olacaktır. Öncelikle bu projedeki bilgileri bir txt dosyasına yazdırıyor, kaydediyor ve verileri o dosya üzerinden çekiyorum. Aslında bu projede pekiştirmek istediğim konu dosya üzerinden veri işlemleri mantığıdır. Bu sistem kullanıcıya 6 adet seçenek sunmaktadır. bunları anlatmam gerekirse: 1- Add Task. Bu seçenek sayesinde ismi ve bitiş tarihi ile yapmam geerekn görevleri giriyorum.2- Show all Task. Bu seçeneğin hedefi kayıtlı tüm görevleri listelemektir. 3- Delete Task. Bu seçeneğin hedefi sistemden silinmesi istenilen görevi silmektir. görevin adı ile bu işlem gerçekleşir. 4- Save Tasks. Bu seçenek sayesinde girilen tüm görevler dosyaya kaydedilir. 5- Load Tasks. Bu görev sayesinde herhangi bir dosyadan görevler yüklenebilir. (kodda dosyanın adı tanımlanması gerkmketedir). 6- EXIT. kullanıcının sistemden çıkış yapmasını sağlar.



# My main goal in this project is to add the tasks that I need to do during the week to a system and to be able to view them from that system and then delete them. Of course, many systems already exist to do this task. However, it would be better to use the system I made during this long journey in my education. First of all, I print the information in this project to a txt file, save it and pull the data from that file. In fact, the subject I want to reinforce in this project is the logic of data operations over the file. This system offers the user 6 options. If I had to explain them: 1- Add Task. Thanks to this option, I enter the tasks that I need to do with their name and due date. 2- Show all Task. The goal of this option is to list all registered tasks. 3- Delete Task. The goal of this option is to delete the desired task from the system. This action occurs with the name of the task. 4- Save Tasks. With this option, all entered tasks are saved to the file. 5- Load Tasks. Thanks to this task, tasks can be loaded from any file. (the name of the file must be defined in the code). 6- EXIT. Allows the user to log out of the system.



# Mein Hauptziel in diesem Projekt ist es, die Aufgaben, die ich während der Woche erledigen muss, einem System hinzuzufügen und sie von diesem System aus anzeigen und dann löschen zu können. Natürlich gibt es bereits viele Systeme, um diese Aufgabe zu erledigen. Es wäre jedoch besser, das System zu verwenden, das ich während dieser langen Reise in meiner Ausbildung gemacht habe. Zunächst drucke ich die Informationen in diesem Projekt in eine TXT-Datei, speichere sie und ziehe die Daten aus dieser Datei. Tatsächlich ist das Thema, das ich in diesem Projekt vertiefen möchte, die Logik der Datenoperationen über die Datei. Dieses System bietet dem Benutzer 6 Optionen. Wenn ich sie erklären müsste: 1- Aufgabe hinzufügen. Dank dieser Option gebe ich die Aufgaben, die ich erledigen muss, mit ihrem Namen und Fälligkeitsdatum ein. Das Ziel dieser Option ist es, alle registrierten Aufgaben aufzulisten. 3- Aufgabe löschen. Das Ziel dieser Option ist es, die gewünschte Aufgabe aus dem System zu löschen. Diese Aktion erfolgt mit dem Namen der Aufgabe. 4- Aufgaben speichern. Mit dieser Option werden alle eingegebenen Aufgaben in der Datei gespeichert. 5- Aufgaben laden. Dank dieser Aufgabe können Aufgaben aus jeder Datei geladen werden. (der Name der Datei muss im Code definiert werden). 6- AUSGANG. Ermöglicht dem Benutzer, sich vom System abzumelden.
